## Epic

**Epic Name**: platform - identity-service

**Description**:
This initiative delivers a platform-aligned identity service for soorma-core with explicit two-tier tenancy boundaries, secure principal lifecycle management, and phased JWT rollout. It modernizes authentication and trust handling while preserving existing integration contracts during coexistence.

The scope includes shared auth-context evolution, identity core domain capabilities, SDK JWT integration, and final legacy-header cutover. The resulting architecture supports secure incremental delivery, clear auditability, and extension-ready trust models for delegated issuers.

**Acceptance Criteria**:
- Platform tenant onboarding and principal lifecycle capabilities are defined with testable acceptance criteria.
- JWT issuance and delegated trust validation contracts are defined and traceable to requirements.
- FR-11 phased rollout is decomposed into mergeable units with explicit dependency order.
- Security and QA validation expectations are attached to each unit.
- Inception artifacts provide enough detail to begin construction unit-by-unit.

**Initiative Artifacts**: `aidlc-docs/platform/identity-service/`

## Stories

### Story: uow-shared-auth-foundation: shared auth context foundation

**Description**:
Implements the shared authentication dependency foundation that introduces JWT-capable context resolution while preserving existing DI/router call sites during coexistence. This unit creates the compatibility backbone required for downstream identity and SDK work.

**Acceptance Criteria**:
- Given coexistence mode, when requests use JWT, then shared dependencies resolve canonical auth context successfully.
- Given coexistence mode, when requests use legacy headers, then compatibility adapter translation preserves existing route call-site behavior.
- Shared dependency contracts expose policy hooks for mandatory and optional claim validation.
- Unit tests verify fail-closed behavior for missing/invalid auth context.

**Linked Epic**: Epic: platform - identity-service

**Dependencies**: None

### Story: uow-identity-core-domain: identity core domain

**Description**:
Implements tenant onboarding, principal lifecycle, token issuance, delegated issuer registration, and claim-context policy inside the new identity domain service. This unit delivers the central business capabilities for platform identity governance.

**Sub-tasks**:
- Implement onboarding and principal lifecycle APIs.
- Implement token issuance with mandatory claim contract.
- Implement delegated issuer trust registration and v1 static-key validation.
- Implement claim mapping and delegated context policy checks.

**Acceptance Criteria**:
- Given a valid onboarding request, when processing completes, then tenant domain and bootstrap principal state are created.
- Given eligible principals, when token issuance is requested, then mandatory claims and tenant policy constraints are enforced.
- Given unregistered delegated issuers, when delegated assertions are submitted, then validation fails safely.
- Identity lifecycle and auth decisions emit structured audit events.

**Linked Epic**: Epic: platform - identity-service

**Dependencies**: uow-shared-auth-foundation: shared auth context foundation

### Story: uow-cutover-hardening: cutover and hardening

**Description**:
Finalizes JWT-only ingress by removing legacy header-only auth behavior and executing hardening checks for security, telemetry, and rollback confidence. This unit closes FR-11 with production-grade cutover readiness.

**Acceptance Criteria**:
- Given JWT-only mode, when legacy header-only requests are sent, then requests are denied safely.
- Cutover unit verifies fail-closed auth behavior and rollback path documentation.
- Security telemetry and audit coverage is complete for deny/allow decisions.
- Build/test instructions include cutover validation checkpoints.

**Linked Epic**: Epic: platform - identity-service
